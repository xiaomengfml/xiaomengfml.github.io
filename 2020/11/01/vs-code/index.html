1 VS Code C/C++ 编程环境安装

1.1 VS Code 安装

​ VS Code 是目前最流行的轻量级编程开发工具，学会恰当使用可以使编程变得高效而有趣。

1.1.1 VS Code 下载

​ 网上搜索 VS Code 关键字 进入 VS Code 主页， 点击蓝色 右上角 Download 按钮，进入 下载页面 选择 System Installer 64 bit 下载，下载完成后打开安装包进行安装。

一路 下一步 完成安装，可以修改安装路径，添加桌面快捷方式。

1.1.2 常用插件安装

​ 活动栏 (A区域) 打开扩展，搜索插件关键字安装

必装插件 其他插件
中文语言包 Atom 外观
C/C++ 插件 多彩括号
七彩边框
…

​ 使用 按快捷键 F1 调出命令面板，输入命令关键字调整。 例如，输入 language 选择 Configurate Display Language 进行语言显示选择

1.2 C/C++环境搭建

1.2.1 gcc编译器安装

方式一（推荐）：离线安装包 直接解压，解压工具 7zip ，将mingw64文件夹放在C盘根目录下。

方式二： 安装 Ming w-w64 。 Architecture 选择 x86_64, 在线下载安装 。

​

​

​ 安装完成后，添加系统变量。首先找到mingw64文件夹，然后将子文件夹bin的路径，比如 C:\mingw64\bin 添加到系统变量，告诉系统我们要用的 gcc.exe 在路径bin下。

​ 我的电脑点右键 弹出 系统 界面

点开 高级系统设置 选择 环境变量， 系统变量 **选择 **Path，点 编辑

空白行处点击，添加刚才 bin 的路径，比如 C:\mingw64\bin 保存。

gcc 配置完成。

以后添加系统变量都是这一个过程。

1.2.2 hello.c (hello.cpp) 程序运行示例

用 VS Code 打开一个文件夹， 资源管理器 中 点击 新建文件 按钮，命名 hello.c

编程 hello.c

  • 编译 打开 终端 菜单，选择 配置默认生成任务， 选择 c/c++: gcc.exe… 自动生成 tasks.json 配置文件。

  • 调试运行 打开 运行 菜单， 选择 添加配置， 选择 C++(GDB/LLDB) 调试器， 选择上面的 gcc 编译配置 自动生成 launch.json 配置文件。 调试运行结果在 终端

编程 hello.cpp

​ 调试运行过程与上面 C 程序 类似， 首先 删除 配置文件夹 .vscode， 重新生成 g++ 编译配置文件和相应运行配置文件。

1.2.3 打开 自动保存

​ 避免每次手动保存，可以打开自动保存，在 **文件 **菜单下 每秒自动保存。

2 Github 托管代码

​ Github 是目前最流行的代码协作平台，也是优秀代码开源平台。学会用好Github

2.1 注册

​ 打开 Github主页，注册过程非常简便，自己操作试试。

​ Github 使用 存储库(或者称为仓库 repository ) 管理 项目，项目里面存放代码及项目相关文档。 点击左上角 logo 显示 存储库列表， 新建 存储库 点击 new 按钮

f’f’f’f’f’f’f’f’f’f’f’f’f’f’f’f’f’f’f

新建存储库 repository，首先是 存储库名称 Repository name 必填， 描述 Description 选填， 存储库类型 公开 Public 还是 私有 Private 选择，初始化存储库可以 勾选 添加 readme 文件 Add a README file 。点击 create repository 按钮 完成 创建。完成后就可以使用存储库管理自己的代码和文档了。

2.2 VS code 使用 Github

2.2.1 Git 安装

​ 安装 Git ，一路 next 。

2.2.2 克隆存储库到本地

进入创建的存储库，点开 code 按钮 ，点击 复制图标，复制 存储库 网址 URL

打开 VS Code，按 F1 调出 命令面板 ，输入 clone ， ctrl + V 复制刚才的网址，

选择 存储库 本地存储位置

下载完成点击打开

2.2.3 上传修改到Github

点击图标 打开 源代码管理 , 首先是点击 暂存 更改过的文件，然后是点击 提交 暂存 到本地 Git，输入提交消息 或者输入空格，再回车， 最后 推送 到Github， 需要时输入Github账号信息。

2.2.4 github账号保存

If you clone GitHub repositories using SSH, then you authenticate using SSH keys instead of a username and password. For help setting up an SSH connection, see Generating an SSH Key.
Tip: You need Git 1.7.10 or newer to use the credential helper.
You can also install a native Git shell, such as Git for Windows. With Git for Windows, running the following in the command line will store your credentials:
$ git config –global credential.helper wincred

3 Typora 课堂笔记和程序文档书写

​ markdown 简洁语法使其成为文档，特别是程序文档的书写语法。typora 是方便的 markdown 书写工具。

3.1 Typora 安装

​ Typora 进入主页下载安装，愉快体验书写的乐趣吧 :smile:

​ 本说明文档用 ==Typora== 完成:astonished:

​

3.2 VS Code 对markdown 支持

​ VS code 原生支持 markdown，可以预览，编辑。建议 VS Code 编写代码， 配合 Typora 写笔记和文档。

​ 右键点击 .md 文档，选择 在资源管理器中显示 再用Typora 打开编辑。

3.3 图床设置

3.3.1 Typora + PicGo

图在 markdown 中是通过链接访问实现的，因此如果想要放置图片供别人访问的话，需要提供公网图片方法连接。

​ typora 设置图片在 文件 菜单中的 偏好设置 图像 选项卡

安装 PicGo(app) 设置好程序安装路径

3.3.2 PicGo + Github

picGo 中可选图床比较多， 以 Github 为例

  1. 新建一个公开的存储库用于存放图片
  2. 点击右上角个人菜单，选中 settings
  3. 弹出的页面 选中左下方 Developer settings
  4. 弹出的页面，选中左下方 Personal access tokens
  5. 点击 Generate new token
  6. 勾选 repo，输入 Note 内容
  7. 保存好 token
  8. PicGo 中设置 仓库名: 用户名/存储库名, 设定分支名: master, 设定Token: 复制刚才保存的Token, 指定存储路径: img/，设定自定义域名: 不填

图片上传成功，图片显示可能会因为网站访问出错而显示失败，可以尝试其他国内图床。
